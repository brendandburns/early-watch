How It Works

Every change goes through a safety check

EarlyWatch intercepts admission requests via a Kubernetes ValidatingWebhookConfiguration and evaluates your ChangeValidator rules before any mutation reaches the cluster.

Features

Built for production Kubernetes

Lightweight, declarative, and easy to extend.

📋
Declarative rules
Define safety policies as ChangeValidator CRs in YAML — no code required. Version-control your policies alongside your application manifests.
🔗
Dependency awareness
The ExistingResources rule type checks for live dependent resources (e.g. Pods matching a Service's label selector) before allowing a destructive operation.
🧩
Expression checks
The ExpressionCheck rule type lets you write simple expressions evaluated against the admission request for custom policy logic.
🎯
Operation targeting
Scope each ChangeValidator to specific operations — DELETE, UPDATE, CREATE — and specific resource types.
💬
Human-readable errors
Every denial surfaces a clear, operator-defined message field so developers instantly know why their operation was blocked.
🔒
Minimal footprint
A single Go binary with no external dependencies beyond the Kubernetes API. Runs as a standard Deployment with least-privilege RBAC.

Custom Resource

ChangeValidator

One CRD. Infinitely composable safety rules.

Rule types

Each ChangeValidator can contain one or more rules. All rules are evaluated; the first violation denies the request.

ExistingResources
Dependency guard. Denies the request when dependent resources still exist — e.g., block Service deletion while matching Pods are running.
ExpressionCheck
Expression guard. Evaluates a simple expression against the admission request object for custom policy logic. 
apiVersion: earlywatch.io/v1alpha1
kind: ChangeValidator
metadata:
  name: protect-service-from-deletion
  namespace: default
spec:
  subject:
    apiGroup: ""        # core API group
    resource: services

  operations:
    - DELETE

  rules:
    - name: no-matching-pods
      type: ExistingResources
      existingResources:
        apiGroup: ""
        resource: pods
        # Use the Service's spec.selector to find Pods
        labelSelectorFromField: spec.selector
        sameNamespace: true
      message: >
        This Service cannot be deleted because
        matching Pods are still running.

CLI Tool

watchctl

watchctl is the EarlyWatch command-line tool. Use it to interact with EarlyWatch from your terminal or CI pipeline.

build

Build watchctl

Compile the CLI from source using the standard Go toolchain.

go build ./cmd/watchctl/...
approve

watchctl approve — sign a resource for an ApprovalCheck rule

The approve subcommand signs a Kubernetes resource's canonical path with your RSA private key and writes the resulting signature as an annotation on the resource. The EarlyWatch webhook verifies the annotation before allowing the guarded operation. 

watchctl approve \
  --private-key /path/to/private-key.pem \
  --group "" \
  --version v1 \
  --resource configmaps \
  --namespace default \
  --name my-config

Run watchctl approve --help to see all available flags.

Getting Started

Up and running in minutes

EarlyWatch requires a Kubernetes cluster (v1.26+), kubectl, and cert-manager for TLS.

1

Install the CRD

Register the ChangeValidator custom resource definition in your cluster.

kubectl apply -f config/crd/bases/earlywatch.io_changevalidators.yaml
2

Create namespace & RBAC

Set up the dedicated namespace and the least-privilege service account.

kubectl create namespace early-watch-system
kubectl apply -f config/rbac/
3

Deploy the webhook server

Deploy the EarlyWatch admission webhook (TLS certificates must be available via cert-manager).

kubectl apply -f config/webhook/
4

Apply a ChangeValidator

Deploy the sample policy that prevents Service deletion while matching Pods are running.

kubectl apply -f config/samples/protect_service.yaml
5

Test it!

Try deleting a Service while its matching Pods are running. EarlyWatch will deny the request with a clear error.

kubectl delete service my-svc
# Error from server: "This Service cannot be deleted because
# matching Pods are still running."
